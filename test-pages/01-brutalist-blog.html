Against Defaults

By Lena Gruber 2026-03-01 1,800 words

Every piece of software ships with defaults. Default fonts. Default colors. Default spacing. Default corner radius. These defaults are not neutral. They are decisions made by someone else, embedded into tools you did not build, shaping output you are responsible for.

When you open a new document in Figma, the canvas is white, the text is black, the font is Inter, and the rectangle has rounded corners. You have made zero decisions but you already have an aesthetic. This is the tyranny of the default. It makes the path of least resistance indistinguishable from a deliberate choice.

The Default is a Decision

Consider border-radius. In CSS, the default border-radius is zero. A rectangle is a rectangle. But design tools have trained an entire generation to reach for 8px corners before asking whether the content demands them. The rounded corner has become a reflex, not a choice.

The same applies to shadows. To gradients. To the particular shade of grey that every design system calls "muted" or "secondary." These are not natural phenomena. They are conventions established by a small number of companies and propagated through their tools and templates.

The question is not whether your defaults are good. The question is whether you chose them.

A designer who accepts every default is not designing. They are assembling. They are using someone else's visual language to package their own content. The result may be functional. It may even be pleasant. But it is not theirs.

Starting From Zero

The alternative is to start from nothing. No template. No component library. No pre-built theme. Open a blank HTML file. Set the font. Set the color. Set the measure. Build the structure from the content outward, making each decision consciously and recording each choice explicitly.

This is slower. It is harder. It produces work that may not match the prevailing aesthetic. Good. The prevailing aesthetic is a product of defaults. Anything that departs from it is, by definition, a decision. And decisions are what design is made of.

The argument is not that defaults are always wrong. Some defaults are excellent. The argument is that you should have to opt into them rather than having to opt out. The burden of proof should fall on the convention, not on the departure.

What Remains

Strip away the defaults and what remains is structure. Content arranged in space according to decisions you made for reasons you can articulate. This is the minimum viable design. Not a style. Not a trend. A discipline of attention applied to every element on the page.

Start with the rectangle. It is the most honest shape a screen can make. Everything else is a choice. Make sure it is yours.
